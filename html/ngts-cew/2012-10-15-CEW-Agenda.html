Navy SLM Pilot and ELT

  • Navy SLM ELT remains on hold waiting for positive test results at ARL and HTL related to SRB 2012 SP2 and the Ssync binary. Ssync_20121010 looks promising but requires a few more days of runtime, and some local-mode walks of the other file systems for further testing.

SRB 2012 SP2, Ssync & SP3

  • SRB 2012 SP2 was installed on the HTL SLM instance on Tuesday, 9/18/2012. and passed the initial suite of tests by Thursday, 9/20/2012.
  • A new integrated client install package was made available on Friday, 9/21/2012.
    • The new versions include a change to address an issue reported by CCAC LiveTime ticket # 140801, RT 40154. This change will allow for GREP_OPTIONS that include '-n'.
  • Multiple Ssync binaries have been delivered since SP2.
    • A new index needs to be created for highest performance with the new Ssync.
  • SP3 currently targeted for release in the end of October time frame.
  • SP2 was installed at ARL on 9/21. The first updated Ssync was installed 9/24. A couple of issues have been discovered.
  • Navy deferred installation of SRB 2012 SP2 and the Ssync until after the end of FY 2012.
  • The new SRB 2012 SP2 client and HPCMP initialization was installed at Navy.
  • GA (Tino) says that within a major release, a newer R3 client should work with the older R3 server, such as Navy has now.
    • An R4 client would not be expected to work with an R3 server.
  • GA (Tino) also says that a client should report an error if it is working with an incompatible server.

SLM Documentation

  • 10/1/2012 CEW call action item: Lawrence will send an email about installing the SLM documentation on an Apache server.
    • An email with subject "SRB 2012 R3 SP2 documentation in html format" was sent to Tracey Smith on 10/4/2012 @ 4:39 Alaska Time.
    • The srb-doc-2012-R3-SP2.tar.gz file includes GA / Nirvana User documentation and HPCMP SLM additional Documentation for the 2012 R3 SP2 release of SRB. The Storage Resource Broker (SRB) documentation is in both html and pdf form. The tar.gz also includes an index to all the documentation.
    • This can be installed on a Linux apache server with the following:
      • cd <somewhere in DocumentRoot>
      • mkdir slm
      • cd slm
      • tar -xzf <path>/srb-doc-2012-R3-SP2.tar.gz
      • chmod -R a+rX .
    • You then need to create a link to the slm directory. If your server does not automatically serve up an index.html for a link to a directory, link to the slm/index.html file directly.
    • Let me know if you need this packaged some other way than a tar.gz file. The file is on Navy systems at: /p/cwfs/murakami/slm-pioneer/srb-doc-2012-R3-SP2.tar.gz because it is 4.4 MB which is a bit large for email.
    • The tar.gz file includes only end user documentation and does not include Admin, Install, SDK, JavaAPI or some of the other documentation in the ${SRB_HOME}/doc (/opt/slm/current/doc on US) folder wherever the SRB client has been installed. This is a stripped down version of what is at: http://people.arsc.edu/~murakami/ngts-cew with a different index.

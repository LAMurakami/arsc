Introduction

Both SAM-QFS and SRB cases can be opened via a single telephone number or web site and are serviced by a single SLM Support Account Manager. SAM-QFS Cases opened through General Atomics (GA) will be recorded in GA's Bugzilla web site and forwarded to My Oracle Support (MOS) on behalf of the user. Once the case is opened with Oracle, the Oracle service request (SR) number will be noted in the Bugzilla issue history. Thereafter, as a minimum, the final resolution and closing date will also be recorded in Bugzilla. All detailed case information can be found on the MOS system. In all cases, it is important that the required information be supplied so that the case can be resolved as quickly as possible.

Bugzilla Search

The GA / Nirvana Bugzilla Web Interface allows rich (and complex) functionality for searching for bugs. The header and footer of each page provides a quick search field that actually creates a search based on what is entered. The header and footer of each page also provides a link to the search page that has Simple search and Advanced Search tabs.

Quick search

Typing just a number in the search box will take you directly to the bug with that ID.

If you put a word or series of words in the search box, Bugzilla will search the Product, Component, Summary, and Comment fields for your word or words. Searching is case-insensitive and Bugzilla is not really searching for words but is searching for strings. For example, searching for "cat" would also find bugs that contain it as part of other words such as a bug mentioning "catagory", "catch" or even "certificate".

The "Status" of "RESOLVED" or "CLOSED" are omitted From the default search of other than just a number.

Simple search

A simple search allows you to select "Status", "Product" and a list of words. Bugzilla will search bug descriptions and comments for those words and return a list of matching bugs sorted by relevance.

Advanced Search

When using Advanced Search the "Product", "Component", "Status", and "Resolution" are always part of the search. The default does not preselect a "Product", "Component", "Status", or "Resolution". If you inadvertently select a value in one of these fields that allow multiple selections you can de-select by using the <control> key and the mouse (the same way you select multiple items).

You can change your default search options by, going to Advanced Search, checking the box for "and remember these as my default search options", then click search

Bugzilla Email Ingestion

Bugzilla has an add-on that allows user to submit new bugs through e-mail. A procmail init script is placed in a user's home directory. This script redirects the bug into a parser and call Bugzilla API to create or update a bug through a defined template. If a user deviates from this template, system will send a reply message with the error message. If a bug submission is successful, system will write a confirmation e-mail.

Prerequisites

  • Must already have a GA / Nirvana Bugzilla account.
    • See Obtaining a Bugzilla Account above.
  • Must set email preferences.
    1. Log in to Bugzilla https://www2.nirvanaware.com/bugzilla with your username and password.
    2. Click Preferences link near the top of the page.
    3. Click Email Preferences tab.
    4. Click Enable All Mail button.
    5. Scroll down to the end of the page and click Submit Changes.

Known Issues (some by design or configuration)

  • Bugzilla cannot handle e-mail replies with quoted or attached previous messages correctly.
    • If not careful, the bug will be populated with the old comments duplicated in the bug comment list.
    • User should clear e-mail body prior to typing a new comment.
  • Bugzilla handles plain/text messages best.
    • If you send a Rich Text (HTML) Only (text/html) email Bugzilla will reject it with the following:
      • Your message did not contain any text. Bugzilla does not accept HTML-only email, or HTML email with attachments.
    • If you send a HTML e-mail (versus regular text e-mail) from a Microsoft email client or through a Microsoft Exchange email server, the bug created may have two additional attachments (one .dat file and .html file. Both of these are your e-mail file).
      • You may delete these manually by logging in to Bugzilla.
    • Bugzilla may handle email sent as "Plain and Rich (HTML) Text" well depending on the client. A message that is sent as a multipart/alternative email with text/plain and text/html parts will have the text/plain part entered as a new comment and the text/html part discarded.
    • I would not trust anything but plain text message that are being sent though a Microsoft Exchange server or Microsoft email client to be handled correctly by Bugzilla.
  • Error messages that user will get may be confusing.
    • The parameter names required for an email bug submission may not match the names reported in errors or the names viewed in the web interface.
      • For example the message received if the required @op_sys parameter is missing is:
        • You must select/enter a OS.
  • The manual says an e-mail signature is ignored but in practice the email signature may be inserted in the bug.
    • To prevent this problem, insert two lines prior to the signature with the first being a blank line and the second containing only '--' (without the quotes).
    • This method does not always seem to work so it is probably best to clear the signature unless you want this in the description or comments of the bug.
  • Features and or parameters found in some documentation may be disabled in this instance of Bugzilla i.e. alias.

  • OS @op_sys Required - The operating system the bug was discovered on. Sample values:
    • Unknown
    • All
    • AIX PPC 32bit
    • AIX PPC 64bit
    • HP-UX PARISC 32bit
    • HP-UX PARISC 64bit
    • Linux IA64 64bit
    • Linux x86 32bit
    • Linux x86 64bit
    • Solaris SPARC 32bit
    • Solaris SPARC 64bit
    • Solaris x86 32bit
    • Solaris x86 64bit
    • Window x86 32bit
  • Hardware @platform Required - What type of hardware the bug was experienced on. Sample values:
    • -- (reads double dashes)
  • @priority (string) Defaulted - What order the bug will be fixed in by the developer, compared to the developer's other bugs. Sample values:
    • Low
    • Normal
    • High
  • @severity (string) Defaulted - How severe the bug is. Sample values:
    • Inquiry
    • Minor
    • Major
    • Critical
  • @cf_itracker_issue_id - Old iTracker issue number.
  • @alias (string) - A brief alias for the bug that can be used instead of a bug number when accessing this bug.
    • Must be unique in all of this Bugzilla.
  • @assigned_to (username) - A user to assign this bug to, if you don't want it to be assigned to the component owner.
  • @cc (array) - An array of usernames to CC on this bug.
  • @qa_contact (username) - If this installation has QA Contacts enabled, you can set the QA Contact here if you don't want to use the component's default QA Contact.
  • @status (string) - The status that this bug should start out as. Note that only certain statuses can be set on bug creation.
  • @target_milestone (string) - A valid target milestone for this product.

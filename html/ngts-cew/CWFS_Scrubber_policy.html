CWFS Scrubber policy

The Center-Wide File System (CWFS) is the panasas high performance shared file system supporting the Utility Server. The largest of the three mount points, $CENTER, is also mounted on all HPC login nodes at a center. Some references to the CWFS in other HEUE documents may be referring to only the $CENTER mount which is roughly 80% of the entire CWFS capacity. A previous document contains more detail on the Center Wide File System Mount Points.

/u/home/$USER = $HOME

All user files in the $HOME directory will be backed up and are not subject to deletion. The total size of files in the $HOME directory will limited by a quota of 10 GB per user.

/u/work/$USER = $WORKDIR

All user files and directory structures in $WORKDIR that have not been modified within 10 days are subject to deletion.

This omits the "accessed/" clause in the BCT Minimum Scratch Space Retention Time Policy Update: 11 June 2009.

The accessed clause will not be honored because this field is not updated on the panasus files system.

/p/cwfs/$USER = $CENTER

All user files and directory structures in $CENTER that have not been modified within 30 days are subject to deletion.

This omits the "accessed/" clause in the BCT Minimum Scratch Space Retention Time Policy Update: 11 June 2009 but uses a much longer retention period.

The $CENTER file system will be defined within SLM as a resource. Users can enroll files on $CENTER in SLM with Sregister and then use Sretain to set Archive attribute and a Retain-Through date. SLM will then control archiving of these files and removal from $CENTER after 30 days. Note that archiving of these files is not immediate but will performed by a SLM daemon at a later time. Deleting a file from $CENTER that has been registered and marked for archive but not yet archived will result in and SLM error when the daemon tries to archive that file and the file will not be archived.

Files that have not been enrolled in SLM with Sregister will be removed from $CENTER by a separate process. This separate process will not delete files that have been enrolled in SLM with Sregister. Note that files placed in SLM with Sput do not make the file on $CENTER known to SLM.

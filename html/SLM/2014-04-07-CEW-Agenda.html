HPCMP BCT Common Set of Environment Variables policy

  • The following statement is repeated in the ARCHIVE_HOST and ARCHIVE_HOME sections in HPCMP BCT Common Set of Environment Variables policy should be corrected or removed:
    • Note this environment variable becomes unnecessary and should be undefined once Storage Lifecycle Management (SLM) goes into production at a center.
  • The repeated statement in the ARCHIVE_HOST and ARCHIVE_HOME sections should/could read:
    • Note the definition and use of this environment variable will change once Storage Lifecycle Management (SLM) goes into production at a center.
    • The combination of the unique values of ARCHIVE_HOME and ARCHIVE_HOST can be used to determine the default resource and location of an individual user's directory on the archival file system that serves a given compute platform. Although the access means will change each individual user will still have a directory on the archival file system that serves a given compute platform.
  • On 12/16/2013 Lawrence stated that we want Don Cable to take this to the BCT.
    • Now that we have the slm module using the ARCHIVE_HOME environment variable we show a clear need for a change of the "this environment variable becomes unnecessary" statement.
  • A module to "properly" set ARCHIVE_HOME at NAVY was created and installed ant NAVY to be loaded prior to the slm module.
    • NAVY sets ARCHIVE_HOME to HOME (/u/home/${USER}) for all users by default.
    • Setting ARCHIVE_HOME to /u/g/${USER} or /u/b/${USER} appropriately does not interfere with with any use of the ARCHIVE_HOME environment variable since users can use either the automounted HOME or the actual file-system directory.
  • On 1/13/2014 Don acknowledged that Lawrence wants the "this environment variable becomes unnecessary" statement removed from the HPCMP BCT Common Set of Environment Variables policy.
  • On 1/27/2014 and 2/3/2014 Don reported no status change on this item.
  • On 3/24/2014 Don stated that he will talk to the BCT about this.

Recent Topics

  • On 3/13/2014 The archive_home module was installed on the Utility Server, Kilrain and Haise at NAVY.
    • See: 3/17/2014 CEW Agenda & Call Notes
  • A new slm module allows cross site authentication. The ARCHIVE_HOME environment variable controls which resource is used as the default.
    • NAVY users must load the new archive_home module prior to using the new slm module for the correct resource to be set as the default.
    • This new module was put in place on ARL systems in December of 2013 and on NAVY systems in March of 2014. It will be put in place at other DSRCs when as part of the SLM Client install.
    • See: 3/17/2014 CEW Agenda & Call Notes
  • Dual SLM and NFS access to archive on HTL US
    • See: 2/10/2014 CEW Agenda & Call Notes
  • The ARL (mcat2x) / MHPCC (mcat5x) / HTL (mcat7x) mcats were not operational on 10/14/2013 due to routing problems introduced in the ARL network during maintenance.
  • On 10/17/2013 Management clarified short file-system names without server are to be used for Slinks and resource namespace.
    • See Email from Tom Kendall with subject: RE: Policy Decision: Collection names underneath (UNCLASSIFIED)
  • The CWFS Scrubber policy is on Centport.
    • The CWFS Scrubber Policy basically says modify time only will be used because access time is not updated on the Panasus files systems. It says this for both ${WORKDIR} and ${CENTER}.
    • The fact that access time is not updated on the Panasus files systems was discussed in the CWFS call on 11/7/2013. Recent work on ${CENTER} scrubbing may not have taken this into consideration.
    • On November 7, 2013 The ${WORKDIR} time was increased from 10 days to 15 days based on 1/12/2012 Change to BCT Policy.
  • As of October 2013 the siauth command that enables PBS transfer queue jobs to authenticate into the STORAGE.HPC.MIL kerberos realm for access to Storage Lifecycle Management has been installed and is functional on the Utility Server, pershing and harold at ARL.
    • The ticket granting ticket has been tested to work with SLM.
    • Only a limited set of user keytabs are currently installed.
    • This functionality will be available to all users once the automated distribution of user keytabs for the STORAGE.HPC.MIL kerberos realm has been implemented.
    • A ${SAMPLES_HOME}/slm/siauth.pbs script has been created on the ARL systems to provide an example of a PBS job using this functionality.
    • The the Utility Server harold and pershing versions are slightly different due to differences in configuration and initialization of PBS jobs on the systems.
    • The sample script has verbose comments and includes some extra output Lawrence thinks is appropriate for sample script.

SLM TOC scheme

The SLM Table Of Contents (TOC) scheme is one of the user accessible schemes. See SgetS for a complete list.

The SLM Table Of Contents (TOC) scheme is created when a user sets attribute values for a Data Object using Sscheme.

The SLM Table Of Contents (TOC) scheme is for Table of Contents attributes of individual members of container files.

Notes and Special considerations for a Multiple Rows Scheme

  • Updating and deleting a particular row of metadata in a multi-row Scheme is supported.
  • The ROW_ID is shown in Sscheme -l starting with SRB 2012 R3.
  • Sls -select is NOT supported for a ROW_ID.
    • The ROW_ID is a relative number that will change if a record of a multi-row Scheme with a lower ROW_ID is deleted.
  • The ROW_ID supported for updating or deleting a particular row of metadata in a multi-row Scheme should be retrieved immediately before the update or delete and if multiple rows (but not all rows) are to be deleted using one set of retrieved information, the rows should be deleted in the order of highest ROW_ID first.
